# v5.5.0 "Terminal Dramático" — Design Spec

**Data:** 2026-04-20
**Autor:** Helbert Moura + Claude (superpowers brainstorming)
**Target release:** v5.5.0
**Predecessora:** v5.1.0 (Terminal Refinado — tokens oklch + MiniMax fixes)
**Tipo:** MINOR (visual redesign profundo, sem breaking de schema)

---

## 1. Visão

Transformar o AI Launcher Pro de "app Tauri genérico com tokens bem feitos" para um **dev tool com identidade forte** — que comunica "launcher de CLIs de IA" já na primeira tela, sem precisar explicar. A v5.1 arrumou a base (oklch, responsividade, fix MiniMax); a v5.5 redesenha a experiência visível.

### Princípios
- **Monospace-first**: display font JetBrains Mono em todas as headings, numbers, labels técnicos.
- **ASCII-adjacent**: bordas, dividers e frames inspirados em TUI (terminal UI) modernas — sem cair no retrô cafona.
- **Hierarquia dramática**: scale contrast real entre title/subtitle/body (não tudo 13-14px uniforme).
- **Motion intencional**: cada transição comunica algo (enter, exit, state change), não enfeita.
- **Dois temas de verdade**: dark e light ambos polidos, cada um com sua personalidade (dark = "servidor às 3h da manhã", light = "papel técnico / blueprint").

### Anti-goals
- Nenhum skeuomorfismo de terminal antigo (scanline grossa, phosphor glow, fundo pixelado).
- Nenhum layout "Bento-by-numbers" — cada superfície tem razão de existir.
- Nenhuma animação "Lottie" / 3D elaborado.


## 5. Iconografia: lucide-react

### Decisão
- Instalar `lucide-react@latest` (~20KB gzipped, tree-shakable)
- Criar `src/icons/index.ts` com **re-exports nomeados** apenas dos ícones usados (preserva tree-shaking)
- Manter `src/presets/PresetIcon.tsx` existente (monogramas SVG da v5.1) — são específicos dos presets, lucide não tem equivalente

### Inventário inicial
```ts
// src/icons/index.ts
export {
  Terminal, Play, Square, Settings, History, DollarSign,
  Package, RefreshCw, Moon, Sun, Command, ChevronRight,
  Check, X, AlertCircle, Download, Upload, Bell,
  Server, Key, Edit3, Trash2, Copy, ExternalLink,
} from 'lucide-react';
```

### Regras
- Default size: `16px`, stroke-width `1.5` (mais refinado que default `2`)
- Usar apenas `currentColor` para stroke (herda do texto)
- Se precisar de ícone que lucide não tem, criar SVG em `src/icons/custom/` com mesmo padrão

## 7. Temas dark + light reais

### 7.1 Separação
Hoje `tokens.css` tem dark como default e aliases pra light ad-hoc. v5.5:
- `src/styles/tokens-dark.css` — paleta dark completa
- `src/styles/tokens-light.css` — paleta light completa

### 7.2 Paletas

**Dark** (existente, ajustado):
- bg: `#0F1012` (antes `#141519`) — mais preto, menos azulado
- accent: esmeralda `oklch(72% 0.15 160)` (mantido)
- text: `oklch(92% 0 0)` → `oklch(50% 0.01 250)` (muted)

**Light** (nova):
- bg: `oklch(98% 0.005 80)` — warm paper, não branco puro
- accent: esmeralda `oklch(35% 0.15 160)` (mais escuro pra contraste AA)
- text: `oklch(18% 0 0)` → `oklch(45% 0.01 250)` (muted)
- border: `oklch(88% 0.005 80)`

Cada superfície verificada nos 2 temas via checklist em `docs/VISUAL_SYSTEM.md`.

## 10. Motion coreografada

### Transições principais
- **Tab switch**: page-enter com stagger dos children (40ms delay cascata)
- **Modal open**: backdrop fade 120ms + dialog scale 0.98→1 em 180ms
- **CLI launch**: card pulsa verde por 300ms ao confirmar
- **Status bar updates**: valor nume antigo crossfade com novo (180ms)

### Princípios
- Max 300ms por transição
- Respeita `prefers-reduced-motion: reduce` (zera tudo)
- Animar só `transform + opacity` (compositor-friendly)
