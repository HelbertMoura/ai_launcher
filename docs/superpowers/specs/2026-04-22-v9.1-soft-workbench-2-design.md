# AI Launcher Pro v9.1 — "Soft Workbench 2.0" Design Spec

**Date:** 2026-04-22
**Version:** 9.1 (visual overhaul)
**Status:** Approved, ready for implementation

---

## 1. Concept & Vision

UI/UX reformulation completa seguindo a direção **Figma-style minimalista moderna**. Cada superfície, interação e estado deve transmitir competência e calma — nada de barulho visual. O usuário abre o app e sente que está usando uma ferramenta profissional, não um demo. Admin unificado: todas as funcionalidades disponíveis para todos os usuários.

---

## 2. Design Language

### Aesthetic Direction
**Figma-style minimalista moderna** — whitespace generoso, superfícies brancas com sombra sutil, ícones line flat em cinza escuro, cor de destaque warm (terracotta/coral) apenas em CTAs e estados ativos.

### Accent Presets (Admin Appearance)
- **Terracotta** (default): `#E07A5F`
- **Coral:** `#E8785F`
- **Amber:** `#D4A017`
- **Sage:** `#7A9E7E`
- **Slate:** `#5C7A9E`
- **Custom:** user-defined via color picker

### Typography
- **Font:** Inter (Google Fonts), fallback: `system-ui, -apple-system, sans-serif`
- **Mono:** JetBrains Mono for paths, commands, code
- **Scale:** 12 / 14 / 16 / 20 / 24 / 32 px
- **Weights:** 400 (body), 500 (medium), 600 (semibold headings)
- **Line-height:** 1.5-1.6 for body, 1.2 for headings
- **Tracking:** -0.02em for headings ≥ 20px

### Spacing System
- **Base unit:** 4px
- **Scale:** 4 / 8 / 12 / 16 / 20 / 24 / 32 / 48 / 64 px
- **Border-radius:** 6px (small), 10px (medium), 16px (large/cards)

### Motion
- **Duration:** 150ms (micro), 250ms (state changes), 350ms (page transitions)
- **Easing:** `cubic-bezier(0.4, 0, 0.2, 1)` — smooth ease-in-out
- **No bounce, no overshoot** — clean, professional

### Visual Assets
**Icons:** Line style, 24x24, stroke 1.5-2px
- Official logos for: Claude, VS Code, Cursor, Gemini, Qwen (from Simple Icons / brand CDNs)
- Custom flat icons for: Kilo Code, OpenCode, Crush, Droid
- Icon color: `currentColor` (adapts to theme)

**No decorative elements** — no gradient blobs, no terminal frames as decoration, no emoji in controls.

## 4. Features & Interactions

### Theme Toggle
- Click: cycles light → dark → light
- Stored in localStorage as `ai-launcher:theme`
- Applied via `data-theme="dark"` on `<html>`
- Transition: 200ms on background/text colors

### Accent Color Picker
- 6 preset circles + "Custom" option
- Click preset: immediate application
- Custom: opens native color picker
- Saved to localStorage as `ai-launcher:accent`

### CLI Cards
- Hover: shadow-md + translateY(-2px)
- Click: selects card (ring in accent)
- Selected state: accent ring + background tint
- Disabled state: opacity 0.5, no hover effect

### Launch Button
- Default: accent background, white text, bold
- Hover: accent-hover background
- Active: scale(0.98)
- Disabled: muted background, no interaction

### Onboarding
- Clean centered card, no TerminalFrame
- Smooth slide-up entrance
- Steps: Welcome → Detect → Auto-detect → Tour → Launch

## 6. Technical Approach

### CSS Architecture
- `tokens.css` — shared CSS custom properties (light base)
- `tokens-dark.css` — dark theme overrides
- `tokens-light.css` — light theme (renamed from current tokens-light.css)
- CSS variables on `:root` for light, `[data-theme="dark"]` for dark

### Theme System
```tsx
// In App.tsx or ThemeProvider
const theme = localStorage.getItem('ai-launcher:theme') ?? 'light';
document.documentElement.setAttribute('data-theme', theme);
```

### Icon System
- Official SVGs downloaded to `public/icons/cli/` and `public/icons/tool/`
- `<img>` tags with `currentColor` filter trick for theme adaptation
- Fallback: simple colored circle with initial letter if SVG missing

### Admin Unification
- Remove `VITE_ADMIN_MODE` env var check
- `adminMode` always true in state
- AdminPanel always accessible via tab or settings

### Build
```bash
# Admin mode always enabled — no special flag needed
npm run tauri build
```
