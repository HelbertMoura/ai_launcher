## 1. Contexto e Motivação

Três problemas convergentes no Launcher e na aba de Tools:

1. **Antigravity CLI não é detectado** mesmo quando instalado. Causa-raiz: a definição em `get_cli_definitions()` aponta para o pacote npm inexistente `@google/antigravity` e para o comando `antigravity`. O CLI oficial se chama `agy` (instalado por script PowerShell em `%LOCALAPPDATA%\agy\bin\agy.exe`).
2. **Gemini CLI descontinuado** continua listado.
3. **Antigravity IDE** é detectado na aba Tools (binário encontrado em `%LOCALAPPDATA%\Programs\Antigravity\Antigravity.exe`) mas **sem versão**, porque Electron apps não respondem a `--version`. A versão real está nos metadados do PE (`ProductVersion = 2.0.0`).
4. **Drag-and-drop** está implementado em `LauncherPage.tsx:74-133` e persiste em `localStorage`, mas o `mousedown` é consumido por `<Button>`/`<Chip>` filhos do `<Card>`, então o drag nunca inicia. Falta também affordance visual.

**Nova função `read_exe_product_version`**:
```rust
pub fn read_exe_product_version(path: &std::path::Path) -> Option<String> {
    let ps = format!(
        "(Get-Item -LiteralPath '{}').VersionInfo.ProductVersion",
        path.to_string_lossy().replace('\'', "''")
    );
    let (ok, out) = run_silent("powershell", &["-NoProfile", "-Command", &ps]);
    if !ok { return None; }
    out.and_then(|s| extract_version(&s))
}
```

## 6. Riscos e Mitigações

| Risco | Mitigação |
|---|---|
| `read_exe_product_version` chama PowerShell e adiciona ~100–300ms à varredura de tools | Cache na sessão (já há padrão similar em `NPM_LATEST_CACHE`). |
| `extra_paths` quebra serialização de `CliInfo` antiga em localStorage | Campos com `#[serde(default)]`; sem impacto frontend porque a fonte da verdade é o backend. |
| Handle ⋮⋮ pode ficar invisível em tema claro | Usar `var(--text-dim)` que já se ajusta ao tema. |
| Script de instalação PowerShell exige permissão de execução | Mantemos o `iwr ... \| iex` (mesma forma usada pelo droid). Usuário aprova como sempre. |
| Flag `--yolo` para `agy` não validada com `agy help` durante o brainstorming | Durante a implementação rodar `agy help` e ajustar `flag` se necessário. Se `agy` não tiver flag de skip-permissions, deixar `flag: None`. |
