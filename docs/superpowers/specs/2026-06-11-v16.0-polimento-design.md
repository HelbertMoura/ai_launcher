## Frente 1 — Agent Analytics

Evoluir a aba Costs para "Analytics". O `TabId` interno permanece `"costs"` (evita churn em storage, palette e atalhos); apenas os labels de UI mudam via i18n.

### Dados

Zero mudança no backend Rust. `UsageEntry` (de `read_usage_stats`) já fornece `date`, `cli`, `provider`, `model`, `tokens_in`, `tokens_out`, `cost_estimate_usd`, `project`.

Novo módulo `src/features/costs/analytics.ts` com funções puras:

- `dailySeries(entries, days = 30)` → `{ date, tokensIn, tokensOut, costUsd }[]` — preenche dias sem uso com zeros (série contínua para o gráfico).
- `byProject(entries, topN = 8)` → ranking de custo/tokens por projeto; resto agregado em "outros".
- `byModel(entries)` → breakdown por modelo.
- `trend(entries, days = 30)` → custo do período atual vs período anterior de mesmo tamanho → `{ currentUsd, previousUsd, deltaPct | null }` (null quando período anterior é zero — evita divisão por zero).

Todas testadas em vitest (incluindo: entries vazias, datas fora da janela, project/model null).

### Gráficos (SVG próprio, zero dependência)

Novos componentes em `src/ui/charts/`:

- `AreaChart.tsx` — série temporal (área preenchida + linha), eixo X com marcas de data esparsas, tooltip nativo via `<title>` por ponto, `role="img"` + `aria-label` descritivo.
- `BarList.tsx` — barras horizontais com label, valor formatado e percentual; usado para top projetos e por modelo.

Tematização exclusivamente via CSS vars do Theme Foundry (`--accent`, `--ok`, `--text-dim`, superfícies). ~200-300 linhas no total. Sem canvas, sem lib.

### Layout da CostsPage

De cima para baixo:

1. Cards de resumo: custo 30d, tokens 30d (in/out), tendência ±% vs período anterior.
2. `AreaChart` de custo/dia (30 dias).
3. Duas `BarList` lado a lado: top projetos e por modelo.
4. `BudgetDashboard` existente (intocado).

i18n: chaves novas adicionadas em `pt-BR.ts` PRIMEIRO (fonte do tipo `Dictionary`), depois `en.ts`.

## Frente 2 — Agent Inbox

### Store

`src/features/inbox/inboxStore.ts`, persistido via `src/lib/storage` (registry zod existente):

```ts
{
  id: string;            // chave de dedup estável
  type: "session" | "budget" | "update" | "doctor";
  title: string;
  body: string;
  ts: number;
  read: boolean;
  targetTab: TabId;      // navegação ao clicar
}
```

Regras:

- **Dedup por chave estável**: ex. `update:claude:2.1.0`, `budget:anthropic:2026-06`, `doctor:path-check`, `session:<session_id>`. Reemitir chave existente atualiza `ts` e re-marca como não lida; nunca duplica.
- **Doctor anti-spam**: notifica apenas na transição ok→falha de cada check (estado anterior guardado no próprio store).
- **Cap de 50 eventos** — os mais antigos saem primeiro (apenas lidos são descartados primeiro; se todos não lidos, descarta o mais antigo).
- API: `pushEvent(evt)`, `markRead(id)`, `markAllRead()`, `clearAll()`, `useInbox()` (hook com `useSyncExternalStore`), `unreadCount()`.

### Fontes de eventos

1. **session-ended** — no handler existente de `useSessionEvents` (App.tsx): título com CLI + status (ok/erro/detached) + duração; `targetTab: "history"`.
2. **budget** — após fetch de usage, comparar com limites por provider (mesma lógica da CostsPage, extraída para função compartilhada); 1 evento por provider+mês; `targetTab: "costs"`.
3. **update** — quando o check de updates detecta versão nova de CLI; dedup por cli+versão; `targetTab: "updates"`.
4. **doctor** — falha em check de diagnóstico (transição); `targetTab: "doctor"`.

### UI na TopBar

- Botão sino com badge numérico de não-lidos (esconde em 0; "9+" acima de 9).
- Dropdown ancorado: lista dos eventos (mais recente primeiro), ícone por tipo, timestamp relativo, indicador de não-lido.
- Acessível: `aria-haspopup`, `aria-expanded`, fecha com Esc, click-outside, itens navegáveis por teclado (setas + Enter), foco retorna ao sino ao fechar.
- Clique no item → `setActive(targetTab)` + `markRead(id)` + fecha painel.
- Rodapé do painel: "marcar tudo como lido" e "limpar".

Testes vitest: dedup, cap, transição do doctor, unreadCount, persistência (mock do storage).

## Frente 3 — Épico de acessibilidade AA

Executado **inline** (não via workflow — mudanças de CSS de tema já se perderam 2x em workflows; bug-057/059).

1. **Pagar a dívida de contraste**: corrigir tokens dimmed/muted nos **7 temas** (dark, light, glacier, amber, phosphor, midnight, high-contrast) para contraste ≥4.5:1 contra as superfícies onde são usados (high-contrast mira ≥7:1). Ajustes mínimos de lightness — preservar o caráter de cada tema.
2. **Trava de regressão**: estender `src/theme/theme-contract.test.ts` com verificação programática de contraste (parse das CSS vars + cálculo de luminância relativa WCAG) para pares token-superfície críticos (`--text-dim` vs `--surface`/`--bg`, `--ok/--warn/--err` vs superfícies).
3. **Reativar `color-contrast`** no teste axe e2e (`e2e/launcher.spec.ts`) — remover a exclusão da regra.
4. **axe no CI**: job no `.github/workflows/quality.yml` que instala browsers do Playwright e roda o spec de a11y (o `webServer` do playwright.config já sobe o Vite).
5. **Command Palette — ARIA combobox**: `role="combobox"`, `aria-expanded`, `aria-controls`, `aria-activedescendant` no input; lista com `role="listbox"`, opções `role="option"` + `aria-selected`; anúncio de resultados.
6. **h1 por página**: cada uma das 11 abas com exatamente um `<h1>` (estilo visual atual preservado — só semântica).
7. **Foco**: retorno de foco ao fechar o palette e o painel do Inbox.
